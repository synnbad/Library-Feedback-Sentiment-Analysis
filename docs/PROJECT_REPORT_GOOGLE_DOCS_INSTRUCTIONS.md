# Instructions for Creating Google Docs Version of AMIA Project Report

## Overview

Your AMIA-formatted project report has been created in 6 parts. Follow these instructions to combine them into a properly formatted Google Doc.

## Files to Combine (in order):

1. `AMIA_PROJECT_REPORT_PART1.md` - Title, Abstract, Introduction
2. `AMIA_PROJECT_REPORT_PART2.md` - Methods (NLP Techniques)
3. `AMIA_PROJECT_REPORT_PART3.md` - Methods (Statistical Analysis, Evaluation)
4. `AMIA_PROJECT_REPORT_PART4.md` - Results
5. `AMIA_PROJECT_REPORT_PART5.md` - Discussion
6. `AMIA_PROJECT_REPORT_PART6.md` - Conclusion, References, Appendices

## Step-by-Step Instructions:

### Step 1: Create New Google Doc

1. Go to https://docs.google.com
2. Click "Blank document"
3. Title it: "Library Assessment DSS - AMIA Project Report"

### Step 2: Set Up AMIA Formatting

**Page Setup:**
- File → Page setup
- Margins: 1 inch all sides
- Paper size: Letter (8.5" x 11")

**Font:**
- Select all (Ctrl+A / Cmd+A)
- Font: Times New Roman
- Size: 12pt
- Line spacing: Double (2.0)
- Alignment: Left (justified for abstract)

### Step 3: Copy Content

1. Open each PART file in order
2. Copy the content
3. Paste into Google Doc
4. Apply formatting as described below

### Step 4: Format Title Page

**Title** (centered, bold, 14pt):
```
AI-Powered Library Assessment Decision Support System: 
A Multi-Source Data Integration Approach Using Natural 
Language Processing and Machine Learning
```

**Authors** (centered, 12pt):
```
[Your Name], [Degree]¹, [Team Member 2], [Degree]¹, [Team Member 3], [Degree]¹
```

**Affiliation** (centered, 11pt):
```
¹[Your Institution], [City], [State], [Country]
```

### Step 5: Format Abstract

- **Heading**: "Abstract" (centered, bold)
- **Text**: Italicized, justified alignment
- **Length**: 125-150 words (currently ~150 words - perfect!)
- **No indent** on first line

### Step 6: Format Main Sections

**Major Headings** (left-aligned, bold):
- Introduction
- Methods
- Results
- Discussion
- Conclusion

**Subheadings** (left-aligned, bold, may be slightly smaller):
- System Architecture
- Data Collection and Preprocessing
- NLP Techniques Implemented
- etc.

**Sub-subheadings** (left-aligned, italic or bold-italic):
- Baseline Method - TextBlob
- Enhanced Method - RoBERTa
- etc.

### Step 7: Format Tables

**Table 1: Machine Learning Models Comparison**
- Insert → Table
- Format with borders
- Header row: bold, shaded background
- Caption above table: "Table 1. Comparison of machine learning models..."

**Table 2: Sentiment Analysis Performance**
- Similar formatting
- Include all metrics

### Step 8: Format Lists

**Bulleted lists:**
- Use standard bullets
- Indent consistently
- Single spacing within list, double between lists

**Numbered lists:**
- Use 1., 2., 3. format
- Consistent indentation

### Step 9: Format Code Blocks

For code examples (if including):
- Font: Courier New or Consolas
- Size: 10pt
- Background: Light gray
- Border: Optional

### Step 10: Format References

**Heading**: "References" (centered, bold)

**Format** (Vancouver style):
```
1. Devlin J, Chang MW, Lee K, Toutanova K. BERT: pre-training of deep bidirectional transformers for language understanding. arXiv preprint arXiv:1810.04805. 2018.

2. Reimers N, Gurevych I. Sentence-BERT: sentence embeddings using siamese BERT-networks. arXiv preprint arXiv:1908.10084. 2019.
```

**Rules:**
- Numbered list
- Hanging indent (0.5 inches)
- Only first word of title capitalized
- Journal abbreviations
- No period after URL

### Step 11: Add Page Numbers

- Insert → Page numbers
- Bottom center or top right
- Start from page 1 (title page)

### Step 12: Add Figures (if needed)

**Figure 1: System Architecture Diagram**
- Insert → Drawing or Image
- Caption below: "Figure 1. System architecture showing..."
- Reference in text: "(Figure 1)"

**Figure 2: Sentiment Analysis Comparison**
- Bar chart comparing TextBlob vs. RoBERTa
- Caption below

### Step 13: Final Checks

- [ ] Total length: 8-10 pages (currently ~10 pages - perfect!)
- [ ] Abstract: 125-150 words ✓
- [ ] All sections present ✓
- [ ] References formatted correctly
- [ ] No orphan headings (heading at bottom of page)
- [ ] Consistent formatting throughout
- [ ] Page numbers present
- [ ] Spell check completed
- [ ] Grammar check completed

### Step 14: Export to PDF

1. File → Download → PDF Document (.pdf)
2. Save as: "Library_Assessment_DSS_AMIA_Report.pdf"
3. Open PDF to verify formatting
4. Check that all content is present
5. Verify page breaks are appropriate

## Content Customization

### Replace Placeholders:

1. **[Your Name]** → Your actual name
2. **[Degree]** → Your degree (e.g., MS, PhD candidate)
3. **[Your Institution]** → Your university name
4. **[City], [State], [Country]** → Your location
5. **Team Member Names** → Actual team member names
6. **GitHub URL** → Your actual repository URL (if applicable)

### Add Your Data:

If you have actual results from your implementation:
1. Replace example numbers with your actual metrics
2. Update tables with your data
3. Add your actual visualizations
4. Include your specific findings

### Adjust Length:

If over 10 pages:
- Reduce appendices
- Condense discussion
- Remove some examples
- Tighten language

If under 8 pages:
- Expand methodology details
- Add more results
- Include more discussion
- Add figures/tables

## Tips for Success

1. **Proofread carefully** - Read aloud to catch errors
2. **Check consistency** - Same terminology throughout
3. **Verify citations** - All references cited in text
4. **Test PDF** - Open on different devices
5. **Get feedback** - Have team members review
6. **Submit early** - Don't wait until deadline

## Common Mistakes to Avoid

1. ❌ Inconsistent heading levels
2. ❌ Missing page numbers
3. ❌ Wrong reference format
4. ❌ Tables/figures without captions
5. ❌ Exceeding page limit
6. ❌ Abstract too long/short
7. ❌ Orphan headings
8. ❌ Inconsistent spacing
9. ❌ Missing sections
10. ❌ Poor PDF quality
