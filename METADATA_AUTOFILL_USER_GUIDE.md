# Metadata Auto-Fill - User Guide

## Quick Start

The metadata auto-fill feature saves you time by automatically detecting and filling in FAIR/CARE metadata fields based on your uploaded dataset.

## Step-by-Step Guide

### Step 1: Upload Your CSV File
1. Navigate to the "Data Upload" page
2. Click "Choose a CSV file" and select your dataset
3. Select the dataset type (survey, usage, or circulation)
4. Enter a dataset name (or use the auto-suggested name)

### Step 2: Auto-Fill Metadata
1. Click the **"Auto-Fill Metadata"** button
2. Wait a moment while the system analyzes your dataset
3. You'll see a success message: "Metadata auto-filled! Review and edit as needed."

### Step 3: Review Auto-Generated Fields
The system will populate these fields:

**Title**
- Automatically formatted from your filename
- Example: `library_feedback_spring_2024.csv` → "Library Feedback Spring 2024"

**Description**
- Includes dataset statistics and characteristics
- Example: "Survey dataset with 150 records and 8 fields. Contains textual feedback responses. Includes rating/score data. Date range: 2024-01-15 to 2024-03-20."

**Source**
- Detected from filename patterns
- Examples:
  - Files with "qualtrics" → "Qualtrics Survey"
  - Files with "alma" or "ils" → "Integrated Library System (ILS)"
  - Files with "google" → "Google Forms"
  - Others → "CSV Upload"

**Keywords**
- Auto-generated from column names and content
- Example: "survey, feedback, ratings, students, 2024"

### Step 4: Edit as Needed
- All auto-filled fields are editable
- Add or modify any information
- The auto-fill is just a starting point!

### Step 5: Add Manual Fields
These fields require manual input:
- **Usage Notes**: Context and guidance for data reuse
- **Ethical Considerations**: Restrictions or ethical guidelines

### Step 6: Upload
Click "Upload Dataset" to save your dataset with the metadata.

## What Gets Auto-Detected?

### From Filename
- **Title formatting**: Removes `.csv`, replaces underscores/hyphens with spaces, applies title case
- **Source detection**: Identifies survey platforms and systems (Qualtrics, ILS, Google Forms)
- **Year extraction**: Detects year patterns in filename

### From Dataset Content
- **Record count**: Total number of rows
- **Field count**: Number of columns
- **Content type**: Identifies feedback, ratings, visits, circulation data
- **Date range**: Extracts earliest and latest dates from date columns
- **User types**: Detects student, faculty, staff mentions

### From Column Names
Keywords are generated by analyzing column names for:
- Feedback indicators: `feedback`, `comment`, `response`
- Rating indicators: `rating`, `score`, `satisfaction`
- User types: `student`, `faculty`, `staff`
- Activity types: `visit`, `session`, `checkout`, `borrow`
- Material types: `book`, `material`, `item`

## Tips for Best Results

### Filename Best Practices
- Use descriptive filenames: `library_feedback_spring_2024.csv` instead of `data.csv`
- Include the source: `qualtrics_survey.csv` or `alma_circulation.csv`
- Include time period: `_spring_2024` or `_march` or `_2024`
- Use underscores or hyphens: `library_feedback` not `libraryfeedback`

### Column Name Best Practices
- Use clear, descriptive column names
- Include keywords like `feedback`, `rating`, `date`, `user_type`
- Be consistent across datasets

### When to Use Auto-Fill
✓ **Use auto-fill when:**
- You have many datasets to upload
- Your filename and columns are descriptive
- You want consistent metadata formatting
- You need a quick starting point

✗ **Skip auto-fill when:**
- Your dataset has sensitive or unusual characteristics
- You need very specific custom metadata
- Your filename is generic (like `data.csv`)

## Editing After Auto-Fill

All fields remain fully editable after auto-fill:

1. **Refine the title**: Make it more specific or formal
2. **Enhance the description**: Add context about collection methods
3. **Correct the source**: If auto-detection was wrong
4. **Add keywords**: Include additional relevant terms
5. **Add usage notes**: Explain intended use cases
6. **Add ethical considerations**: Document any restrictions

## Troubleshooting

**Q: The auto-fill button doesn't work**
- Make sure you've uploaded a CSV file first
- Check that you've selected a dataset type
- Verify the file is a valid CSV

**Q: I get an encoding error when using auto-fill**
- The system automatically tries multiple encodings (UTF-8, Latin-1, CP1252, UTF-16)
- If auto-fill fails, you can still fill in metadata manually
- For best results, save your CSV with UTF-8 encoding:
  - Excel: File → Save As → "CSV UTF-8 (Comma delimited)"
  - Google Sheets: File → Download → CSV (automatically UTF-8)
  - LibreOffice: File → Save As → Select "Unicode (UTF-8)" encoding

**Q: The auto-detected metadata is wrong**
- No problem! All fields are editable
- Simply modify the incorrect fields
- The auto-fill is just a suggestion

**Q: Some fields are empty after auto-fill**
- This is normal - some metadata can't be auto-detected
- Fill in the remaining fields manually
- Usage notes and ethical considerations always require manual input

**Q: Can I use auto-fill on previously uploaded datasets?**
- Not currently - auto-fill only works during initial upload
- You can edit metadata later using the "Edit Metadata" button

**Q: My file has special characters (é, ñ, ü, etc.)**
- The system automatically handles multiple character encodings
- Special characters should work fine in most cases
- If you see � or garbled text, try re-saving with UTF-8 encoding
